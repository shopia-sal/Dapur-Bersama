
##Software Requirements untuk Website Resep Makanan Online

##1. Pendahuluan

Dokumen ini menjelaskan kebutuhan perangkat lunak untuk website resep makanan online. Tujuannya adalah untuk memberikan panduan yang jelas mengenai fitur, fungsionalitas, dan spesifikasi teknis yang diperlukan untuk pengembangan sistem.

##2. Kebutuhan Fungsional

Kebutuhan fungsional adalah fitur dan fungsi yang harus ada dalam sistem untuk memenuhi kebutuhan pengguna.

###a. Pengguna (Customer)
- **Pendaftaran dan Login** : Pengguna harus dapat mendaftar dan membuat akun, serta login untuk mengakses fitur-fitur website.
- **Pencarian Resep** : Pengguna dapat mencari resep berdasarkan kategori, bahan, atau nama resep.
- **Menampilkan Resep** : Pengguna dapat melihat daftar resep lengkap dengan informasi seperti bahan, langkah-langkah, waktu memasak, dan foto.
- **Ulasan dan Rating** : Pengguna dapat memberikan ulasan dan rating pada resep yang telah dicoba.
- **Menulis Resep** : Pengguna dapat menulis dan mengunggah resep mereka sendiri, lengkap dengan foto dan langkah-langkah.

##b. Kebutuhan Non-Fungsional
Kebutuhan non-fungsional adalah karakteristik sistem yang tidak terkait langsung dengan fungsi tetapi penting untuk kinerja dan pengalaman pengguna.

###1.Kinerja
- **Waktu Muat Halaman** : Halaman harus dimuat dalam waktu kurang dari 3 detik.
- **Kapasitas Pengguna** : Sistem harus mampu menangani minimal 500 pengguna aktif secara bersamaan.

##2. Keamanan
Autentikasi: Sistem harus menerapkan autentikasi pengguna yang aman, termasuk penggunaan kata sandi yang kuat dan autentikasi dua faktor (2FA).

##3. Usability
Antarmuka Pengguna: Antarmuka harus intuitif dan mudah digunakan, dengan navigasi yang jelas.

##4.Kebutuhan Perangkat Lunak
Kebutuhan perangkat lunak mencakup semua perangkat lunak yang diperlukan untuk pengembangan dan operasional sistem.

###a. Sistem Operasi
- Windows 10 atau lebih baru untuk pengembangan lokal.
- macOS versi terbaru untuk pengembang yang menggunakan perangkat Apple.
- Linux (Ubuntu 20.04 LTS) untuk server produksi.

###b. Basis Data
- MySQL untuk menyimpan data terstruktur seperti pengguna, resep, dan ulasan.
- MongoDB untuk menyimpan data yang tidak terstruktur atau semi-terstruktur.

###c. Framework dan Bahasa Pemrograman
- Backend: PHP (menggunakan framework seperti Laravel atau CodeIgniter).
- Frontend: JavaScript 

###d. Alat Pengembangan
- IDE: Visual Studio Code atau PHPStorm untuk pengembangan.

##5. Integrasi
Sistem harus dapat terintegrasi dengan layanan pihak ketiga untuk meningkatkan fungsionalitas:

-API Media Sosial: Untuk memungkinkan pengguna membagikan resep di platform media sosial.
